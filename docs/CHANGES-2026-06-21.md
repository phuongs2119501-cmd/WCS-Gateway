## 1. The canonical map was re-pointed to the real PLC  ⭐ most important
**File:** `WcsSystem/Wcs.PlcService/Config/db500-keys.json` (v2 → **v3**)
- Every offset corrected to the live layout from main's `Data Mapping PLC/DataPlc1.cs`.
- Shifted +4..+8 bytes from the shuttle region on: barcode `DBB32→DBB36`, barcode OK/NG `46→50`,
  done `48→52`, system state `50→54`, crane pos `54→58`, shuttle pos `58..→66..`, gate `70→78`,
  heartbeat `74→82`.
- **New real-PLC fields added** (new keys, old keys kept so services still compile): `reqAutoRun`,
  `reqStop`, full `crane2*` state+position, `plcFail`, `gateExport`, `directionBlock2`.
- `minLengthBytes` 76 → 86.
- ⚠️ **Provisional** — to be confirmed when we meet the gateway developer (per PO rule 1).

**Generated from it (our own tool, not Gemini):**
- `WcsSystem/Wcs.PlcService/Plc/Db500Map.g.cs` — regenerated via `tools/Db500MapGen`.
- Regen command: `dotnet run --project tools/Db500MapGen`.

## 2. Simulator re-pointed + shuttle motion added
**File:** `WcsSystem/Wcs.PlcService/Plc/SimS7Backend.cs`
- Now derives **every** offset from `Db500Map` — **zero literal byte offsets** remain, so the sim can
  never drift from the real map again.
- Added crane **+ shuttle** (X/Z/B) motion during a job (previously only the crane moved).
- DB image covers DBW84 (length 128 ≥ 86).

## 3. Response contract — two versions (PO decision)
**File:** `WcsSystem/Wcs.PlcService/Controllers/HomeController.cs`
- `GET /status` (our nested shape) — **unchanged**, still `StatusController`.
- `GET /api/status` — upgraded to the **owner-compatible flat shape** (matches main's `SendDataAPI`:
  `gateImport1`, `done1`, `fail1`, `directionBlock2_Plc1`, …). One `SystemState`, two read-only views.
- Removed a duplicate `/api/status` route to fix an ambiguous-match error.

## 4. Bug fixes caught by live testing (PO rule 4: "don't be wrong like Gemini")
**File:** `WcsSystem/Wcs.PlcService/Worker.cs`
- Heartbeat was hardcoded to the **old** offset `"DB500.DBX74.0"`. After the v3 shift that byte lives
  inside `shuttle2Z` (`DBW74`), so the heartbeat silently corrupted shuttle-2's Z (read 257 vs 1).
  Fixed to `Db500Map.wcsHeartbeat` (`DBX82.0`). This is the exact literal-offset class of bug we are
  eliminating — TC1's `shuttle2.z=1` check now guards against it.
